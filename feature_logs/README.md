# Feature request log

One file per feature request, named `NNN_short-name.md` in chronological order.

## Format

```markdown
# NNN — Short title

**Requested:** YYYY-MM-DD HH:MM PDT
**Status:** complete | in progress | blocked

## Request
Verbatim quote of what the user asked for.

## Actions taken
- Files changed
- Key implementation details
- Config added (if any)

## Verification
How it was tested / left for user to verify.

## Notes
Anything noteworthy, gotchas, follow-ups.
```

Each new feature request from the user gets its own file. The agent adds the file when the request comes in and updates the **Actions taken** section as work progresses.
